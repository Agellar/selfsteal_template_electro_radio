# ElectroWave Radio 🎧

Одностраничный сайт-витрина онлайн-радиостанций **электронной музыки** на русском языке.
House, Techno, Trance, Drum & Bass, Synthwave, Ambient и другие жанры — в прямом эфире, прямо в браузере, без регистрации и трекеров.

> Чистый **HTML + CSS + Vanilla JS**, без фреймворков и сборки. Просто откройте файл в браузере.

---

## ✨ Возможности

- **43 радиостанции** электронной музыки (Radio Record, Retrowave.One, Discover Trance), сгруппированные по жанрам.
- **Постоянный мини-плеер** внизу экрана: play/pause, переключение станций, громкость, статус эфира.
- **Развёрнутый модальный плеер** с большим визуализатором, текущим треком, историей и кнопкой «Поделиться».
- **Визуализатор звука** на `<canvas>`: настоящий спектр через Web Audio API, а при отсутствии CORS — плавная симуляция эквалайзера (64 полосы).
- **Интеграция с системой (MediaSession)** — станция и трек на экране блокировки, реакция на аппаратные кнопки play/pause/вперёд/назад.
- **Авто-переподключение** при обрыве потока: ретраи с экспоненциальной задержкой (1→8 с) и индикатор «Переподключение…».
- **Фильтр по жанрам** (теги-пиллюли), **поиск** по названию (дебаунс 300 мс), тоггл **«Только онлайн»** и **сортировка** (по умолчанию / сначала онлайн / А–Я / по жанру).
- **Избранное** ★ на карточках с фильтром «Избранное».
- **Реальные логотипы станций** Radio Record (best-effort, с фолбэком на монограмму).
- **Статус онлайн / офлайн** для каждой станции (ленивая проверка через `Audio` + таймаут 8 с).
- **Текущий трек** для станций Radio Record — опрос API каждые 30 секунд (best-effort, тихо отключается при CORS-ограничениях).
- **История прослушивания** — последние треки хранятся **только в памяти вкладки**.
- **Светлая и тёмная темы**: по `prefers-color-scheme` + ручной переключатель.
- **Опциональное сохранение настроек** (тема, громкость, избранное) — выключено по умолчанию, включается тумблером в подвале (privacy-first).
- **Свои станции с вашего сервера** — плейлист из аудиофайлов (играют по очереди и зацикливаются) или живой поток, через простой `stations.json` без правки кода.
- **Адаптивный дизайн** mobile-first (375 → 1440 px), sticky-плеер не перекрывает контент.
- Информационные разделы: что такое электронная музыка, ключевые субжанры, фестивали и площадки, иконические треки, главные артисты.
- Доступность: один `<h1>`, корректная иерархия заголовков, видимые фокус-кольца, `aria`-атрибуты, поддержка `prefers-reduced-motion`.

## 🚀 Запуск

Проект полностью статический — сборка не нужна.

**Вариант 1. Открыть файл напрямую**

```
electrowave-radio.html → двойной клик / открыть в браузере
```

**Вариант 2. Локальный сервер** (рекомендуется — корректная работа Web Audio API и проверки статусов)

```bash
# любой статический сервер, например:
npx serve .
# или
python3 -m http.server 8000
```

Затем откройте `http://localhost:8000/electrowave-radio.html`.

> ⚠️ Для воспроизведения нужен доступ в интернет: аудиопотоки отдают сторонние серверы (Radio Record и др.).
> Потоки по HTTP (например, Discover Trance) на HTTPS-страницах блокируются браузером как *mixed content* — для них показывается соответствующее сообщение.

## 🛠 Технологии и решения

| Область | Реализация |
| --- | --- |
| Стилизация | CSS-переменные, цвета в **OKLCH**, текст через `clamp()`, шаг отступов 4px |
| Шрифты | `Space Grotesk` (заголовки) + `Inter` (текст) через Google Fonts |
| Аудио | Один глобальный `Audio`-элемент, переключение источников «на лету», авто-переподключение с backoff |
| Интеграция с ОС | Media Session API (метаданные + аппаратные кнопки) |
| Хранилище | `localStorage` только при явном согласии (тема, громкость, избранное); по умолчанию выключено |
| Визуализатор | Web Audio API (`AnalyserNode`, 64 полосы) с безопасным откатом на симуляцию |
| Анимации | Только CSS-переходы и `requestAnimationFrame` (без GSAP и сторонних библиотек) |
| Появление секций | `IntersectionObserver` (scroll-reveal), `content-visibility: auto` |
| Текущий трек | Публичный API Radio Record, опрос раз в 30 с |
| Логотип | Оригинальный inline-SVG (волна-сигнал в форме буквы «E») на `currentColor` |

### Про CORS и визуализатор

Web Audio API требует CORS-доступа к аудиопотоку. При первом запуске приложение **однократно проверяет** возможность использования `crossOrigin = 'anonymous'`:

- если поток отдаёт CORS-заголовки — рисуется **настоящий спектр**;
- если нет — приложение навсегда отключает маршрутизацию через Web Audio (чтобы звук **гарантированно играл**) и показывает **симулированный эквалайзер**.

Так воспроизведение работает всегда, а визуализатор анимируется во время проигрывания в любом случае.

## 🎚 Свои станции (с вашего сервера)

Можно добавить собственные станции — как **плейлист из аудиофайлов** на вашем сервере, так и **живой поток** (Icecast/Shoutcast). Код сайта менять не нужно.

1. Скопируйте `stations.example.json` в **`stations.json`** и положите рядом с `electrowave-radio.html` — сайт подхватит его автоматически. Либо передайте адрес параметром: `electrowave-radio.html?manifest=https://ваш-сервер/stations.json`.
2. Опишите станции. **Плейлист из файлов** — поле `tracks`:

```json
{
  "stations": [
    {
      "name": "Моя синтвейв-подборка",
      "genre": "Synthwave",
      "tags": ["Synthwave"],
      "description": "Мои треки.",
      "icon": "https://ваш-сервер/cover.png",
      "tracks": [
        { "title": "Night Drive", "artist": "Me", "src": "https://ваш-сервер/music/1.mp3" },
        { "title": "Neon",        "artist": "Me", "src": "https://ваш-сервер/music/2.mp3" }
      ]
    }
  ]
}
```

   **Живой поток** — вместо `tracks` укажите `stream`:

```json
{ "name": "Моя станция", "genre": "Techno", "stream": "https://ваш-сервер/live" }
```

Плейлист-станция: треки играют по очереди и **зацикливаются**, показываются точные названия; в развёрнутом плеере доступны список треков (клик — перейти) и кнопки предыдущий/следующий трек. Недоступный файл автоматически пропускается.

**Требования к вашему серверу:**
- **HTTPS** — если сайт открыт по https, ссылки в манифесте тоже должны быть https (иначе браузер заблокирует mixed-content).
- **CORS** (опционально) — заголовок `Access-Control-Allow-Origin` на файлах включает настоящий спектр-визуализатор; без него воспроизведение работает, но эквалайзер рисуется симуляцией.
- **Форматы** — те, что понимает браузер: mp3, aac, ogg.

## 🔒 Конфиденциальность

ElectroWave Radio спроектирован приватным по умолчанию:

- **не собирает персональные данные**, не требует регистрации;
- **не использует cookie**, аналитику и сторонние трекеры;
- история прослушивания живёт только в памяти открытой вкладки и удаляется при её закрытии;
- **сохранение настроек — по выбору пользователя**: по умолчанию ничего не пишется на диск. Тумблер «Запомнить мои настройки» включает запись темы, громкости и избранного в `localStorage` только этого браузера; выключение тумблера немедленно удаляет эти данные. `sessionStorage` и cookie не используются в любом режиме;
- аудиопотоки отдают сторонние сервисы; на эти соединения распространяются их собственные политики;
- для пользователей из РФ обработка персональных данных не ведётся (соответствие **152-ФЗ**).

Полный текст доступен в самом сайте: подвал → «Политика конфиденциальности».
